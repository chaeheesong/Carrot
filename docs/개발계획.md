## ⚠️ 와이어프레임 확인 후 변경/충돌 사항 (먼저 읽기)

와이어프레임을 반영하며 기존 계획 대비 아래를 조정했다:

1. **[확정] 카테고리 = 포함** (2026-07-17 사용자 재확인).
   와이어프레임 홈 피드의 카테고리 칩(`전체 / 디지털기기 / 가구·인테리어 / 생활가전 / 유아동`) 기준. Product에 `category` enum 필드 + 피드 필터 구현.
2. **[확정 일치] 거래완료 상태 = 포함.** 상세 화면에 "거래완료" 회색 배지 존재. `status`(SELLING/SOLD) 유지.
3. **[조정] 피드 카드의 찜 = 토글 버튼 없음.** 카드엔 "찜 N"(개수)만 표시. 하트 토글 버튼은 **상품 상세**에만 존재. → 목록 즉시 토글은 범위에서 제외.
4. **[확정] 플랫폼 = 데스크톱 웹 우선.** 와이어프레임이 넓은 화면 기준(피드 5열 그리드, 채팅 2단 레이아웃). 반응형은 축소 대응만.
5. **[미포함] 상품 등록 화면은 와이어프레임에 없음.** 기능은 필요하므로 자체 폼으로 구현하되 시안 없이 진행(추후 시안 오면 맞춤).

## 디자인 스펙 (와이어프레임에서 추출)

**색상**
- Primary(오렌지): `#FF7E36` 계열 — 버튼, 링크, 내 말풍선, 안읽음 뱃지, 활성 탭 밑줄
- 배경: 크림/베이지 `#F5F0EA` 계열, 카드/패널은 흰색
- 카드 썸네일 placeholder: 파스텔(베이지·민트·블루) 톤
- 텍스트: 진회색 `#333`, 보조 메타는 연회색
- 선택된 칩/배지: 검정 배경 흰 글씨(전체 칩), 회색 배지(거래완료)

**타이포**
- 제목/가격 굵게(Bold), 메타("3분 전 · 찜 4")는 작고 연하게
- 가격은 카드에서 눈에 띄게 강조

**공통 컴포넌트**
- 라운드 큰 카드/패널(radius 12~16), 부드러운 그림자
- 오렌지 풀-width 버튼(로그인/채팅하기/전송)
- Pill 형태 필터 칩

## 화면별 요구사항 (와이어프레임 6p)

### P1. 로그인 (`/login`)
- 중앙 카드: 타이틀 "동네장터" + 서브 "우리 동네 중고거래"
- 이메일 / 비밀번호 입력
- **인라인 에러 메시지**: "이메일 또는 비밀번호가 올바르지 않습니다." (필드 하단, 연분홍 배경)
- 오렌지 풀-width "로그인" 버튼
- (회원가입 링크 — 와이어프레임엔 없지만 가입 경로 필요, 최소 폼 추가)

### P2. 홈 피드 (`/`)
- 상단 좌: "서울시 강남구" + **"동네변경"**(오렌지 링크)
- 카테고리 칩 행: `전체`(활성=검정) / `디지털기기` / `가구·인테리어` / `생활가전` / `유아동`
- 상품 카드 **그리드(넓은 화면 5열)**: 정사각 썸네일 / 제목 / **굵은 가격** / "N분 전 · 찜 N"
- 카드 클릭 → 상세
- 무한 스크롤(cursor)

### P3. 상품 상세 (`/products/[id]`)
- "← 목록으로" 뒤로가기
- **좌: 이미지 영역** + "이미지 1 / 4" 인디케이터(캐러셀)
- **우: 정보 카드** — 상태 배지("거래완료"), 제목("캠핑 의자"), 가격("18,000원"), 판매자(아바타 + `june_h`), 하트 버튼 + 오렌지 "채팅하기"
- 하단 탭: **"상세설명" / "문의 댓글 (2)"** (활성 탭 오렌지 밑줄)
- 댓글 목록 + 작성

### P4~P5. 채팅 (`/chat`, `/chat/[roomId]`) — 2단 레이아웃
- **좌측 채팅방 리스트**(항상 표시): 아바타 / 이름 / 상품명 / 마지막 메시지 / 시간 / **안읽음 뱃지(오렌지 숫자)**, 선택 시 연주황 하이라이트
- **우측(방 미선택)**: "채팅방을 선택하세요 →"
- **우측(방 선택)**:
  - 헤더: 상대 아바타+이름, 우상단 **상품 미니카드**("아이맥 27인치 / 450,000원")
  - "↑ 이전 메시지 불러오기"(cursor 페이지네이션)
  - 말풍선: 상대=회색 좌측, 나=오렌지 우측
  - 하단 입력창 + 오렌지 "전송"

## 0. 리포지토리 구조 (모노레포)

```
Carrot/
├─ apps/
│  ├─ web/                 # Next.js 14 (App Router) + Apollo Client + Tailwind
│  └─ server/              # Apollo Server + Prisma + graphql-ws
├─ packages/graphql/       # (선택) 공유 스키마/코드젠 타입
├─ pnpm-workspace.yaml
└─ package.json
```
- `apps/web` → Vercel, `apps/server` → Railway (각각 루트 디렉터리 지정으로 빌드).
- Tailwind에 위 디자인 토큰(오렌지 primary, 크림 배경)을 `theme.extend`로 등록.

## 1단계 — 기반 세팅 + 인증 (1~2일)

**백엔드**
- [ ] `apps/server` 초기화(Apollo Server v4, TypeScript, ts-node-dev)
- [ ] Prisma 초기화 → `schema.prisma`(User, Product+`category`+`status`, ProductImage, Like, ChatRoom, Message, Comment)
- [ ] Supabase 프로젝트 → `DATABASE_URL` → `prisma migrate dev`
- [ ] `signup`/`login` mutation(bcrypt, JWT), context에서 토큰 검증 → `context.userId`
- [ ] 공개/인증 리졸버 구분 가드 헬퍼(`requireAuth`)

**프론트**
- [ ] `apps/web` Next.js + Tailwind(디자인 토큰) + Apollo Client
- [ ] ApolloProvider + authLink + errorLink
- [ ] **P1 로그인 화면**(인라인 에러 포함) + 최소 회원가입 폼(동네 select)
- [ ] 토큰 저장 + 라우팅 처리(게스트는 피드/상세 열람 가능)

**완료 기준**: 회원가입 → 로그인(실패 시 인라인 에러) → `me` 확인.

## 2단계 — 홈 피드 (2일)

- [ ] 서버: `products(neighborhood, category, after, first)` cursor 페이지네이션(게스트 호출 허용)
- [ ] 서버: `likeCount` 필드 리졸버(DataLoader로 N+1 방지)
- [ ] 서버: 카테고리 enum + 필터
- [ ] 서버: `toggleLike(productId)` mutation(상세용, 인증)
- [ ] 프론트: **P2 피드** — 동네 헤더/동네변경, 카테고리 칩 필터, 5열 카드 그리드, "N분 전 · 찜 N" 메타
- [ ] 프론트: 무한 스크롤(IntersectionObserver + fetchMore)
- [ ] 시드 스크립트로 더미 상품 20~30개(카테고리/상태 섞기)

**GraphQL 시연 포인트**: 카드에 필요한 필드만 조회 → 오버페칭 없음.

## 3단계 — 상품 상세 + 이미지 업로드 (2일)

- [ ] 서버: `product(id)` — images, seller(+`otherProducts`), comments 중첩 조회
- [ ] 서버: `addComment`, `createOrGetChatRoom`, `createProduct`, `updateProductStatus`(판매자만) mutation
- [ ] 프론트: **P3 상세** — 이미지 캐러셀("1/4"), 정보 카드(상태 배지·가격·판매자·하트·채팅하기), 상세설명/문의댓글 탭
- [ ] 프론트: 판매자 본인이면 "거래완료로 변경" 노출
- [ ] 프론트: 상품 등록 폼 → Supabase Storage 직접 업로드 → URL만 mutation 저장
- [ ] Supabase Storage 버킷 + 정책

**GraphQL 시연 포인트**: 이미지·판매자·다른 상품·댓글을 **1회 요청**으로 → 언더페칭 없음.

---

## 4단계 — 실시간 채팅 (2일, 핵심)

- [ ] 서버: `graphql-ws` WebSocket 서버(`useServer`) + `PubSub`
- [ ] 서버: `myChatRooms`(상대·상품·마지막메시지·안읽음수), `chatRoom(id).messages(cursor)`
- [ ] 서버: `sendMessage` → 저장 후 `messageAdded` publish, `messageAdded(chatRoomId)` subscription(참여자만 인가)
- [ ] 프론트: Apollo `GraphQLWsLink` + `split`(subscription만 WS)
- [ ] 프론트: **P4/P5 채팅** — 2단 레이아웃, 방 리스트(안읽음 뱃지), 방 헤더 상품 미니카드, 이전 메시지 불러오기, 좌/우 말풍선(오렌지=나), 입력+전송
- [ ] 프론트: `subscribeToMore`로 새 메시지 실시간 병합

**GraphQL 시연 포인트**: 두 창에서 새로고침 없이 실시간 도착 → Subscription 강점.

**완료 기준**: 창 2개(구매자/판매자)에서 실시간 대화.

## 리스크 & 메모
- Subscription 인가(참여자만 구독) 빠뜨리기 쉬움 → 4단계 필수 체크.
- N+1: `seller`, `likeCount`, `unreadCount`에 DataLoader.
- 이미지 업로드는 GraphQL multipart 대신 **클라이언트 → Storage 직접 업로드 → URL 저장**.
- 상품 등록 화면은 시안 부재 → 자체 구현 후 시안 오면 조정.
- 반응형: 데스크톱 우선이나 최소한 모바일에서 깨지지 않게 그리드 열 수/채팅 2단 → 1단 전환 대응.
